# 结单导入实现注意事项

这份文档沉淀长桥、盈立、汇丰 PDF 结单导入中的通用规则。后续新增券商导入器时，应优先复用这些语义，避免同一类流水在不同平台上产生不同的资产、收益或去重行为。

## 平台适配范围

- 长桥导入器已经覆盖的交易语义应作为每个证券平台适配的基准清单，包括：证券买入/卖出、期权交易与到期、拆股/合股、IPO/新股、真实入金/出金、融资/融券利息、证券分红与扣税合并、孤立税费、活动奖励/现金奖励、账户内部换汇。
- 新平台适配不能只实现当前样例中出现的类型。即使给定 PDF 样例没有某类流水，也应检查该平台结单是否可能出现该类型，并在交付说明或测试计划中明确列出“样例缺失，尚无法验证”的交易类型。
- 如果样例缺失导致无法可靠实现某类解析，应显式提出需要补充样例或真实脱敏文本；不要默默把该类型当作平台不支持，也不要用默认币种、默认证券代码或普通现金流水来猜测导入。
- 适配完成时应给出覆盖矩阵：长桥基准类型、当前平台是否已实现、是否有样例验证、缺失样例的风险说明。

## 文本提取与清洗

- PDF 文本提取结果不是稳定表格。实现时应同时考虑单行、拆列、多行备注、分页断行和 OCR 误识别。
- 解析前先做 Unicode 规范化、空白裁剪和页眉页脚过滤。常见页码、账户声明、责任说明、表头、汇总行不能进入业务解析。
- 金额解析应保留负号信息用于判断流入/流出，但落库的现金流出金额使用正数，由 `TradeType` 决定资金方向。
- 不要为缺失字段做高风险猜测。特别是现金流水缺少明确币种上下文时，应跳过真实入金/出金，而不是默认 USD/HKD/CNY。

## 币种、市场与上下文

- 现金流水的 `market` 由币种决定：`USD -> Market.US`，`HKD -> Market.HK`，`CNY -> Market.CASH`。
- 平台结单常把币种作为独立标题行，例如 `HKD`、`美元`、`港元`。解析器应维护当前币种上下文，但只在明确标题或行内币种出现后使用。
- 证券交易仍按交易市场解析；不要把现金流水的市场规则反向套到买卖记录上。

## 统一输出结构

- 所有平台导入器最终输出 `ParsedStatementTrade`，让 repository 层统一落库。
- `quantity` 对现金流水固定为 `1.0`，金额放在 `price` 和 `amount`。
- 非证券现金流水使用 `symbol=CASH`；融资/融券利息可使用 `symbol=INTEREST`。
- 非证券现金流水不得触发证券名称自动修复，避免把活动奖励、入金、换汇误识别为股票。

## 去重与幂等

- `tradeRef` 必须稳定，并包含平台前缀、日期、类型、币种、金额和必要的源行序号。
- 同一 PDF 重复导入应依赖 `平台 + tradeRef` 跳过旧记录。
- 分红配税和换汇配腿之后，`tradeRef` 应基于合并后的语义生成，而不是使用单边流水编号。

## 现金流水语义

- `DEPOSIT` / `WITHDRAW` 只表示外部出入金。
- 账户内部换汇使用 `TradeType.FX_CONVERSION`，并填充 `fxFromCurrency/fxFromAmount/fxToCurrency/fxToAmount/fxRate`；换汇不影响现金余额、净入金、总资产和收益。
- 活动礼包、现金奖励等非证券收益使用 `TradeType.OTHER`，按正向现金收入处理，但不进入证券分红统计。
- 手工税费或无法配对的税费仍可使用 `TradeType.TAX`，但证券分红税优先和分红合并。

## 分红与扣税配对

- 先把分红和股息税解析成候选，再按同一日期、同一币种、同一证券代码或相近备注配对。
- 配对成功时只输出一条 `DIVIDEND`：税前分红写入 `price`，扣税写入 `tax`，净入账由计算层使用 `price * quantity - tax` 得出。
- 提取证券代码时优先从备注识别 `NVDA.US`、`TSM(US...)`、`0005.HK` 等格式；提取不到时使用 `CASH`。
- 未配对分红仍输出 `DIVIDEND tax=0`；未配对税费输出 `TAX`，避免丢失流水。

## 换汇配对

- 换汇入账/出账先作为候选解析，再按日期、币种对、汇率和备注配对。
- 成功配对后输出单条结构化 `FX_CONVERSION`。例如 `HKD -> USD`：`fxFromCurrency=HKD`、`fxFromAmount=5000`、`fxToCurrency=USD`、`fxToAmount=636`、`fxRate=0.1272`。
- 只有单边换汇时也输出一条 `FX_CONVERSION`，只填充可识别的一侧，不猜测缺失金额。

## 测试样本维护

- 每个平台至少保留买卖回归样本、现金流水小样本和真实 PDF 回归样本。
- 小样本应覆盖：分红配税、无税分红、孤立税费、活动奖励、真实入金/出金、换汇双腿合并、单边换汇、币种上下文。
- 真实 PDF golden 若因新增支持类型导致记录数变化，应同步更新预期；如果 golden 不在仓库内，至少新增可重复的 `parseText` 单测锁住新语义。
- 计算层需覆盖 `DIVIDEND tax`、`OTHER` 和 `FX_CONVERSION` 的现金、收益、净入金口径。
