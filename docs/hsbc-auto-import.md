# 汇丰自动导入方案

## 目标

让 StockLedger 可以把汇丰证券交易通知自动转换成交易流水，并尽量避免重复入账。

## 路径评估

### 1. 短信通知

- 难度：低到中
- 优点：实时；正文结构稳定；不需要邮箱 OAuth / IMAP；当前样本已经包含方向、代码、价格、数量、交易编号
- 风险：Android 需要 `RECEIVE_SMS` 权限；如果未来要上架 Google Play，需要额外评估 SMS 权限政策
- 结论：最适合作为第一阶段主路径

### 2. 邮件通知正文

- 难度：中
- 优点：字段更完整，通常能拿到股票全名、状态、交易编号；可用于历史补录
- 风险：真正“自动拉取”需要接入邮箱账号、授权、同步策略与安全存储；不同邮件客户端格式也可能不同
- 结论：适合作为第一阶段的辅助导入入口，而不是首个全自动抓取通道

### 3. 电子结单 / 电子通知书附件

- 难度：高
- 优点：最适合月度对账、补全手续费税费、处理部分成交
- 风险：附件可能加密；PDF/HTML 结构更脆；往往不是实时到达；需要额外做解密和版式解析
- 结论：更适合作为第二阶段“补录 + 对账”能力

## 当前选型

第一阶段采用“汇丰短信自动导入 + 短信/邮件正文分享导入”：

- 新收到的汇丰短信：通过 Android 短信广播自动解析并入库
- 历史短信 / 邮件正文：通过系统分享文本到 StockLedger 复用同一解析器导入

这个组合的好处是：

- 自动化程度已经足够覆盖日常成交入账
- 不需要先做邮箱账号托管
- 解析逻辑统一，后续接邮件拉取或 PDF 对账时可以复用交易编号去重

## 已落地的基础能力

- 汇丰短信与邮件正文解析器
- `externalReference` 交易编号字段，用于幂等去重
- `sourceChannel` 来源字段，用于标记短信 / 邮件来源
- 汇丰短信开关与权限入口
- `ACTION_SEND` 文本分享导入入口

## 二期建议

- 增加邮件 IMAP / API 拉取
- 增加 PDF/HTML 结单补录
- 对“部分成交、多次回报、不同成交价”做更细粒度建模
- 对账时补全手续费、税费与成交日期校准
