# 公司行动处理 TODO

## 背景

当前拆股/合股记录仍以普通流水形式保存在各个平台下。例如同一只股票在盈立和长桥都持有时，两个平台都可能各自同步出同一天、同一比例的拆股记录。

这符合平台流水审计直觉，但在跨平台汇总计算时需要特别处理：同一证券市场事件只能对汇总持仓生效一次，否则会出现重复折算。

## 当前策略

短期采用“各平台独立记录，汇总时去重”：

- 单平台视图保留本平台的拆股记录。
- 删除或重导某个平台数据时，不影响其他平台的拆股流水。
- 汇总计算按 `market + symbol + tradeDate + ratio` 识别同一拆股事件，只应用一次。

## 未来可演进方向

长期可以考虑引入独立的“公司行动事件”层：

- 拆股、合股、分拆、派股等市场事件全局唯一保存。
- 平台流水只作为该平台持仓受事件影响的审计记录，或引用全局事件。
- 组合计算统一从公司行动事件层读取，避免各计算入口重复实现去重。
- 导入、手工录入、自动同步都应先匹配现有事件，再决定是否创建新事件。

## 待办

- 评估是否需要新增 `CorporateActionEntity` 或类似数据表。
- 梳理拆股之外的公司行动类型：合股、派股、分拆、代码变更、并购换股。
- 设计平台流水与全局事件的关联方式。
- 统一组合总览、收益分析、收益日历、持仓详情中的公司行动处理入口。
- 为跨平台重复拆股、单平台拆股、删除平台数据后的行为补测试。
